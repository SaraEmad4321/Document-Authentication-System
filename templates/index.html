What happens when you sign

1
SHA-256 hash of the document is computed.
2
Hash is signed with ECDSA using signer's private key.
3
A .sig.json package is saved: document + signature + signer ID + timestamp.

Verification will fail if…

✕
Document was tampered — even 1 changed byte alters the hash.
✕
Wrong public key — verifying with a different user's key.
✕
Signature modified — any change to the .sig.json invalidates it.

Cryptographic design

1
ECDSA over SECP256R1 — Elliptic Curve Digital Signature Algorithm. Provides authentication and non-repudiation using asymmetric key pairs.
2
SHA-256 hashing — fixed-length 256-bit digest. Makes signing large files efficient and detects any modification.
3
No encryption used — documents stay plaintext. This system provides integrity and identity, not confidentiality.
4
Signed package format — filename, signer ID, algorithm info, Base64 signature, and ISO timestamp bundled as JSON.

Security goals

✓
Integrity — SHA-256 hash detects tampering.
✓
Authentication — ECDSA signature binds identity to document.
✓
Non-repudiation — private key ownership cannot be denied.
—
Confidentiality — intentionally not provided. Not required by the threat model.
